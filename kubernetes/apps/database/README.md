# kubernetes/apps/database/

This namespace contains operators and centralized services for databases and in-memory datastores. Instead of deploying distinct database instances manually, this cluster leverages operators to provision, manage, and scale datastores on a per-application basis.

## Application Inventory

| App | Description | Components |
|-----|-------------|------------|
| **cloudnative-pg** | Kubernetes native operator for PostgreSQL cluster management. | `helmrelease` |
| **dragonfly** | Highly performant, multi-threaded Redis-compatible in-memory datastore. | `helmrelease` |
| **opensearch-operator** | Kubernetes operator for provisioning and managing OpenSearch clusters. | `helmrelease` |

## Architecture & Usage

### CloudNative-PG (PostgreSQL)
Stateful applications in the `default` namespace (such as Forgejo, Immich, and Paperless) rely on CloudNative-PG. Rather than using a single monolithic database server, apps define their own `Cluster` resources, and the CNPG operator provisions a dedicated, highly available PostgreSQL cluster tailored to that specific app.

### Storage & Backups
Databases provisioned by these operators integrate with the cluster's core infrastructure:
- **Storage**: Database clusters dynamically provision persistent volumes (PVCs) typically backed by **Rook-Ceph** block storage for high performance and replication.
- **Backups**: Persistent data protection is handled either by CloudNative-PG's built-in S3 backup/WAL archiving mechanisms or via **VolSync** for raw PVC snapshot replication to Kopia/S3.

### Secret Management
Database credentials and connection URIs are either automatically generated by the operators (creating Kubernetes Secrets that applications can mount) or injected via **SOPS-encrypted** cluster secrets during Flux reconciliation.

## Adding a new Database Operator

When adding a new database operator to this namespace, ensure it follows the standard two-level GitOps structure (`apps/database/<app>/ks.yaml` and `apps/database/<app>/app/`) and is added to the namespace's root `kustomization.yaml`.
